CHAPTER 12

风险识别与应对策略

七大风险均已有明确应对方案

⚡钉钉专业版升级

风险：标准版配额无法支撑日常运营
应对：路演明确说明配额预算，体现工程预判能力

🔌MCP 客户端兼容

风险：Claude Desktop 版本差异
应对：Demo 使用稳定版，关键场景预录视频备份

📍签到防作弊

风险：自研防作弊存在边界
应对：HMAC 60s 二维码 + GPS 围栏 + 价值观答题三重校验，深度方案后续迭代

🌐跨企业冷启动

风险：缺少真实合作企业数据
应对：内置 3 家虚拟对手企业数据

🧠LLM API 不稳定

风险：路演现场调用超时
应对：platform/llm 抽象层支持 Claude / DeepSeek / Qwen / OpenAI 四 Provider，配置即切换

💎Go MCP SDK 不成熟

风险：缺少现成轮子
应对：自研 JSON-RPC over SSE，platform/mcp 轻量实现，16 工具复用 service 层

🎯价值观维度配置主观性

风险：不同企业对价值观的定义差异较大，易陷入"配什么维度都不对"
应对：沿用公司核心 4 大维度（客户第一 / 坦诚沟通 / 一号位 / 敢于创新）作为默认模板，企业可后台自定义增删；远期开放"价值观模板市场"，行业最佳实践共享
