CHAPTER 01

项目背景 · 价值观落地的运营难题

"墙上写一套，做事另一套" — 现有共享 Excel 难以承载企业文化运营

📊数据载体

现状：共享 Excel 单点管理

误操作、并发冲突、无审计、负数积分异常无法预防

⚙️业务流程

现状：HR 手工台账加减分

公平性存疑，员工无从核对，规则不透明

🚫规则不闭环

现状：扫码即扣分，无校验

未中奖也被扣，体验差，申诉成本高

📉数据沉淀

现状：仅记录数值，无维度归属

无法回答"哪个价值观最薄弱"，缺乏量化抓手

CORE QUESTION
如何让企业文化从口号变成可观测、可调节、可优化的运营指标？
本质命题：文化积分系统不仅要解决工具问题，更要回答"如何让文化运营具备指标体系"这一战略问题

APPENDIX

价值观维度数据模型

核心三张表：维度配置 · 积分流水（维度记账）· 维度积分快照

-- 价值观维度配置（租户级） CREATE TABLE value_dimensions ( id BIGINT PRIMARY KEY, tenant_id BIGINT NOT NULL, code VARCHAR(32) NOT NULL, -- 维度编码 e.g. customer_first name VARCHAR(64) NOT NULL, keywords VARCHAR(255), -- 关键词 · 用于 AI Prompt 注入 weight DECIMAL(3,2) DEFAULT 1.00, sort_order INT DEFAULT 0, enabled TINYINT DEFAULT 1, UNIQUE KEY uk_tenant_code (tenant_id, code) );
-- 积分流水（维度记账） CREATE TABLE point_transactions ( id BIGINT PRIMARY KEY, tenant_id BIGINT NOT NULL, user_id BIGINT NOT NULL, dimension_id BIGINT NOT NULL, -- 价值观维度 amount INT NOT NULL, -- 积分变动（正负） activity_id BIGINT, reason VARCHAR(255), operator_id BIGINT, created_at TIMESTAMP, INDEX idx_user_dim (user_id, dimension_id), INDEX idx_tenant_dim_time (tenant_id, dimension_id, created_at) );
-- 员工维度积分快照 -- 用于雷达图与排行榜 CREATE TABLE user_dimension_scores ( user_id BIGINT NOT NULL, tenant_id BIGINT NOT NULL, dimension_id BIGINT NOT NULL, total_score INT DEFAULT 0, quarter_score INT DEFAULT 0, year_score INT DEFAULT 0, updated_at TIMESTAMP, PRIMARY KEY (user_id, dimension_id) );
🎯设计要点
  • 维度配置租户级：每个企业可独立配置
  • 流水必带维度：保证每分都可追溯
  • 快照表加速查询：雷达图 / 排行榜性能保障
  • keywords 字段：直接喂给 LLM Prompt
