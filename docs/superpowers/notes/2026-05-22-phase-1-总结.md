## 关键决策与变更（与原 plan 的对照）

| 变更项 | 原因 |
|--------|------|
| pnpm `9.12.0` → `11.2.2` | 本机 corepack 默认版本，已补 sha512 摘要 |
| @react-three/fiber `8.x` → `9.6.1`，drei `9.114.0` → `10.7.7` | React 19 适配 |
| mysql `8.4` → `8.4.4` | 本机镜像缓存 |
| `packages/ui/src/index.ts` 改用显式 `.ts`/`.tsx` 后缀 | ESM 严格模式（Node 不支持裸目录导入）|
| `packages/ui` 新增 `"./uno"` 独立导出路径 | vite.config.ts 在 Node ESM 下加载 UI 时无法处理 `.tsx`，需隔离 uno 配置与 React 组件 |
| `@storybook/react` pin `^8.6.0` | 与 react-vite 同步版本 |
| values/repository 集成测试新增 `TestMain` | 原测试直接 TRUNCATE 但未建表，需先 drop+recreate DB + migrate |
| `main.tsx` `getElementById('root')!` 改为 null-check + throw | biome `noNonNullAssertion` 规则 |
